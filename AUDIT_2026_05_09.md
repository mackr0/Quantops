### Issue 3 — `/api/dashboard-totals` 500s; happy-accident hides it; fixing one half alone would break the dashboard
**Status:** 🛠️ in progress
**Source:** page audit; `views.py:4761` and `:4775`

**Deep-dive verification (2026-05-09):**
- ✅ The audit's claim that `get_today_total` doesn't exist in `ai_cost_ledger.py` is correct (only `log_ai_call` and `spend_summary`). Reproduced via direct python.
- ❌ The audit's headline "permanently $0" was **wrong**. Live `journalctl -u quantopsai-web` shows the endpoint actually 500s every 5 minutes with `ImportError: cannot import name 'build_context' from 'user_context'` at `views.py:4761` — BEFORE control ever reaches the `get_today_total` import.
- The JS at `dashboard.html:181-194` reads `d.error` and returns early; nothing gets overwritten. Server-rendered initial values (which use the correct `spend_summary(...)["today"]["usd"]` path at `views.py:411` and the jinja footer at `dashboard.html:157`) stay on screen all day. **User has been seeing correct values precisely because the endpoint is broken.**
- Risk of fixing only one of the two broken imports: endpoint would start returning `{cost_today: 0}` for every profile, JS would start writing `$0` over the correct cells. **Must fix both together** — exactly the regression the user warned against.

**Plan:**
- Fix both broken imports in `views.api_dashboard_totals`:
  - `from user_context import build_context` → `from models import build_user_context_from_profile` (line 4761)
  - `from ai_cost_ledger import get_today_total; cost_today = float(get_today_total(...))` → `from ai_cost_ledger import spend_summary; cost_today = float(spend_summary(ctx.db_path)["today"]["usd"] or 0)` (lines 4775-4776)
- Add regression test that mocks `spend_summary` + `build_user_context_from_profile`, calls the endpoint, asserts the JSON returns the mocked cost value (not 0).
- Verify on prod: hit the endpoint with auth, confirm 200 + non-zero cost; tail journald for 5 min and confirm the 500s stop.

**Outcome:** _in progress_
